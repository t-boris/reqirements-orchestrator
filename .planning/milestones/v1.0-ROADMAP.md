## Overview

Build a Slack bot that acts as a proactive business analyst, driving thread-based conversations to gather requirements and create Jira tickets. The journey moves from infrastructure (database, LLM) through core agent logic (ReAct loop, skills) to integrations (Slack, Jira) and finally deployment.

## Milestone Summary

**Key Decisions:**
- Thread-scoped sessions (one thread = one ticket)
- ReAct loop over rigid phases
- PostgreSQL only (no Zep)
- Direct Atlassian API (no MCP)
- Gemini as default LLM
- Socket Mode over HTTP endpoints
- Persistent background event loop for async in sync handlers

**Issues Resolved:**
- AsyncPostgresSaver initialization pattern
- Gemini response parsing (list content blocks)
- Event loop binding for asyncio locks
- Postgres version compatibility on deploy

**Issues Deferred:**
- Conversation history fetching (v1.1)
- Improved onboarding UX (v1.1)

**Technical Debt Incurred:**
- None significant
